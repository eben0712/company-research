# 上市公司深度覆盖报告 Skill

## 触发后的交互流程

触发skill后，弹出AskUserQuestion确认窗口让用户选择，包含三个选择题 + 一个自由输入框：

**问题1：分析深度**
- 快速：1轮搜索，2-3千字概览
- 标准（默认）：3轮搜索，5-8千字深度报告
- 深度：4轮搜索 + 一手资料验证，8-12千字完整报告

**问题2：关注重点**
- 全面覆盖（默认）：走完所有11个模块，自动按公司类型调整权重
- 侧重商业模式：深化模块1-4（公司概况/商业模式/行业/护城河），轻财务
- 侧重竞争格局：深化模块4-5（护城河/竞品生态位），加强生态位对比
- 侧重财务分析：深化模块7-9（财务趋势/盈利预测），详拆分产品分区域数据
- 其他（自由描述）

**问题3：信息准确度偏好**
- 快速：WebSearch汇总券商研报+财经新闻为主，关键数据存疑时补读原文
- 准确（默认）：优先读取年报/招股书原文关键章节，再用WebSearch补充行业和竞品信息

**其他需求（自由输入框）**：任何特殊要求，如"重点看海外对标""跳过财务""对标日本XX公司"等。

用户选择完毕后，直接开始搜索和写作，不再二次确认。

## 核心原则

1. **客观穿透**：亮点和风险同等权重。每个正面判断都审视反方。数据必须可溯源——年报/招股书（巨潮资讯网 cninfo.com.cn、东方财富网 eastmoney.com）> 券商研报 > 权威财经媒体。没有数据时标注"公开信息有限，以下为推测"。

2. **三层剥离**：不只描述"卖什么"，穿透到"真正赚什么钱"。表层（消费者看到的）→ 中层（中间商看到的）→ 底层（真实利润引擎）。

3. **生态位思维**：不孤立分析，始终放在竞争生态中定位。区分真壁垒（时间/工艺/规模/网络效应）和假壁垒（先发优势但可复制/渠道红利但不可控）。

## 公司类型判定与权重调整

读 `references/company-type-guide.md`，在搜索前快速判定公司类型：

- **渠道型**（如锅圈）：核心矛盾在加盟生态/门店密度/供应链效率 → 权重在模块2/5/6
- **产品型**（如有友）：核心矛盾在品牌心智/品类天花板/渠道控制力 → 权重在模块2/4/5
- **制造型**（如海天/安井）：核心矛盾在规模效应/成本优势/产能利用率 → 权重在模块2/3/7
- **平台型**（如美团）：核心矛盾在网络效应/双边生态/take rate → 权重在模块2/4/5
- **内容/IP型**（如完美世界、阅文集团、光线传媒）：核心矛盾在IP生命周期、爆款持续性、内容pipeline厚度 → 权重在模块2/4/7

用户选"全面覆盖"时按类型自动调整权重。选"侧重XX"时按用户指定的来。详细判定指南见 `references/company-type-guide.md`。

## 信息来源与搜索策略

### 一手资料来源
- A股年报/招股书：巨潮资讯网（cninfo.com.cn）→ PDF原文；东方财富网（eastmoney.com）→ 财务数据
- 港股：港交所披露易（hkexnews.hk）
- 用WebSearch搜索"公司名 年报/招股书 关键数据"即可获取精确数字
- 准确模式下优先Read年报PDF关键章节（发行人基本情况/业务与技术/管理层讨论/财务数据）

### 三轮搜索执行
详细关键词库见 `references/search-guide.md`。

**第一轮（5-8个WebSearch并行）**：公司发展历程/创始人/股权结构、商业模式、行业规模/增速、竞争格局。准确模式下先读年报关键章节再搜索。

**第二轮（3-5个WebSearch并行）**：自有工厂 vs ODM代工比例、各渠道产品差异化、财务精确数据（分产品/分区域/分渠道）、竞品一对一深度对比。

**第三轮（2-3个WebSearch）**：国际对标案例、最新风险事件、关键数据交叉验证。

## 分析框架：11个模块

### 模块1：公司概况与发展历程

一句话定位 + 三阶段发展史 + 股权结构表（含治理关注点）+ 近5年关键财务概览表（营收/利润/毛利率/净利率）。

### 模块2：商业模式三层拆解

```
表层生意（消费者看到的）：XXX
中层生意（中间商看到的）：XXX
底层生意（真正的赚钱逻辑）：XXX
一句话定位：XX本质是一家XX，而非XX
```

追问：收入来自谁？利润源于制造/品牌溢价/渠道差价？自产 vs ODM代工比例？自有工厂数量和产能？

### 模块3：行业分析

市场规模与增速（引用弗若斯特沙利文/Euromonitor/券商研报等权威来源）+ 细分赛道结构图 + 行业集中度（CR5）+ 消费趋势判断。

### 模块4：护城河评估

真壁垒 vs 假壁垒逐项分析 + 四维度星级评分表（品牌心智/工艺技术/规模成本/渠道网络）+ 一句话总结最核心壁垒。

### 模块5：竞品生态位分析

竞争对手三梯队表 + 生态位象限图（至少两个维度）+ 国际对标公司对比表（至少5个对比维度）+ 与最直接竞品一对一深度对比。

### 模块6：渠道与产品矩阵

渠道-产品差异化表格 + 各渠道毛利率对比 + 渠道控制力评估（自有门店/经销商/外部平台）。

> **行业适配**：如分析对象为游戏/互联网/金融等非实体商品公司，应将此模块语义替换为
> "产品生命周期与分发渠道"（如游戏各产品流水/生命周期阶段/各平台收入拆解）、
> "用户获取与变现渠道"（如买量 vs 自然量/各渠道获客成本）等，保留"各维度收入结构
> 差异分析"的核心意图，不机械套用"经销商/门店"框架。

### 模块7：新业务/第二曲线

机会与挑战分列 + 可行性评级 + 历史新业务孵化成功率。

### 模块8：财务趋势与盈利质量

近5年核心指标趋势表（至少8个指标×5-6个时间段）+ 分产品/分区域/分渠道收入与毛利率 + 增长质量判断（量价齐升/以价换量）+ 风险信号标注（应收激增/现金流恶化/毛利率连降）。

### 模块9：盈利预测

**第一层（重点）**：定性趋势判断 + 核心假设及理由 + 关键变量敏感性分析。
**第二层（参考附录）**：基于假设的2-3年营收/利润估算表。必须标注"以下为基于公开信息的参考估算，非精确预测"。

### 模块10：风险提示

至少5个公司特有的具体风险。每个格式：
```
风险名称（短期/长期结构性）
- 触发条件：XXX
- 影响程度：XXX
- 当前信号：XXX
```
避免"行业竞争加剧""宏观经济波动"等万能模板风险。

### 模块11：总体判断与可持续性评估

- "这个商业模式到底能不能持续"的明确结论
- 乐观情景 vs 悲观情景双情景分析
- 可持续需要满足的关键条件 + 崩溃的触发条件
- 一句话最终判断

## 报告输出

### Markdown 初稿（一次性输出全部模块）

严格按 `references/report-template.md` 的精确模板。标题格式：
```
# 公司名（股票代码）：[对仗式标题概括核心逻辑]
```
保存到用户桌面：`[公司名]_深度分析报告.md`（根据当前系统环境确定实际桌面路径）。

### 用户审阅修改

一次性展示初稿后等待反馈。用户可集中标注所有问题后一次性修改，也可逐模块修正。

### Word 导出

用户确认内容后，找到本 skill 目录下的 `scripts/export_to_docx.py`，运行：
```bash
python <skill_dir>/scripts/export_to_docx.py "<桌面路径>/[公司名]_深度分析报告.md" "<桌面路径>/[公司名]_深度分析报告.docx"
```
`<skill_dir>` 为本 skill 的根目录，`<桌面路径>` 根据用户实际系统确定。

## 重要约束

- **不编造数据**：搜不到就标注"公开信息有限，以下为推测"
- **不回避风险**：龙头公司也要客观指出风险，不做PR稿式的全面看好
- **不套模板**：根据公司核心矛盾调整模块权重，不同公司分析重点不同
- **对标要具体**：国际对标必须给出数据对比和模式差异分析
- **财务优先一手**：年报 > 招股书 > 券商研报 > 财经新闻
- **中文输出，简洁直接**：下明确判断，用类比帮助理解（如"类似白酒老窖池""像优衣库SPA模式"）
- **路径不硬编码**：Markdown/Word 输出路径、脚本路径均根据用户实际系统动态确定，不从 SKILL.md 复制固定路径
- **关键词按行业适配**：搜索时根据公司所处行业选择对应关键词（见 `references/search-guide.md` 行业适配章节），不机械使用消费品默认关键词
